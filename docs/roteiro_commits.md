# Roteiro de commits semânticos

Faça cada commit somente depois de concluir e testar a alteração correspondente. Não crie
commits vazios apenas para atingir a quantidade mínima.

1. `docs: cria README com objetivo escopo e metodologia`
2. `chore: cria estrutura inicial de diretórios`
3. `feat: adiciona autenticação de demonstração`
4. `feat: implementa banco e cadastro de tarefas`
5. `feat: adiciona listagem e painel por status`
6. `feat: implementa edição exclusão e atualização de status`
7. `style: cria interface responsiva do TaskFlow`
8. `test: adiciona testes das regras de tarefa`
9. `test: adiciona testes das rotas web`
10. `ci: configura testes e flake8 no GitHub Actions`
11. `docs: adiciona diagramas UML e roteiro do Kanban`
12. `feat: adiciona prioridade crítica e filtro de tarefas`

Dica: vincule o commit da mudança de escopo ao card ou issue correspondente.
