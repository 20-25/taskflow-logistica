# Roteiro do GitHub Projects — Kanban

Crie um Project em formato **Board** e deixe as colunas **To Do**, **In Progress** e **Done**.
Use os cards abaixo. A distribuição sugerida serve para produzir um print completo; ajuste os
status conforme o trabalho realmente avançar.

## To Do

1. Revisar documentação final e prints.
2. Gravar vídeo pitch de até quatro minutos.

## In Progress

3. Validar o sistema no navegador.
4. Conferir execução do GitHub Actions.

## Done

5. Criar README com objetivo, escopo e metodologia.
6. Criar estrutura `/src`, `/tests` e `/docs`.
7. Implementar login de demonstração.
8. Implementar cadastro e listagem de tarefas.
9. Implementar edição, exclusão e atualização de status.
10. Criar testes automatizados com Pytest.
11. Configurar pipeline de CI no GitHub Actions.
12. **Mudança de escopo:** adicionar prioridade crítica e filtro.

O card 12 deve estar visível porque comprova a mudança de escopo explicada na Parte 1.
